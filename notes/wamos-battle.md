# WAMOS BATTLE DESIGN NOTES

## Game State Init

players (addresses)
player ids
players wamo ids (after staking)
wamos data

## Game State Ongoing

game status
turn count
wamos status
timestamps (start, last turn)

## Game process

1. create game, invite sent
2. players connect wamo, stake
3. game begins when start conditions met
4. turns rotate
5. a players wamos are defeated or player resigns
6. glory and spoils are awarded, outcome recorded
7. wamos unstaked, transferred back to owning player
